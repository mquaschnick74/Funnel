# iVASA Content Production System

Content system for iVASA — a voice-first depth psychology platform serving people in therapy deserts. This system manages X (Twitter) content production for Phase 1 marketing: Problem Awareness.

## Mission Context

158 million Americans live in therapy deserts. iVASA exists for them. Our content does not introduce iVASA as a solution yet — we are building audience recognition of the crisis first. The solution reveal comes later, through bio, pinned posts, and replies.

## Node Map

### Brand Nodes (always load)
- [[brand/voice.md]] — Tone, warmth, sophistication, what we sound like
- [[brand/forbidden.md]] — Words and phrases never permitted in any post
- [[brand/apple-principle.md]] — Lead with WHY, not WHAT. The core structural rule.

### Platform Nodes (load based on destination)
- [[platforms/x.md]] — X/Twitter-specific rules: format, character limits, advocacy tone, image-first openings

### Audience Nodes (load based on topic)
- [[audience/therapy-desert.md]] — The access crisis: statistics, human reality, emotional texture

### Content Nodes (always load)
- [[content/hooks.md]] — The opening image rule. The diagnostic. What a good first line looks like.
- [[content/pca-insight.md]] — Depth psychology differentiation: what we do that no one else does

## Execution Instructions

When generating an X post:
1. Read [[brand/voice.md]] — internalize tone before writing a single word
2. Read [[brand/forbidden.md]] — eliminate prohibited language
3. Read [[brand/apple-principle.md]] — confirm you are leading with WHY
4. Read [[platforms/x.md]] — apply platform-specific format rules
5. Read [[content/hooks.md]] — test your opening line against the image diagnostic
6. Read [[audience/therapy-desert.md]] — ground the post in human reality
7. Output one post, ready to publish. No preamble. No explanation. Just the post.
