# iVASA Content Production System

Content system for iVASA — a voice-first depth psychology platform serving people in therapy deserts. This system manages X (Twitter) content production using a 9-slot PCA pattern rotation.

## Mission Context

158 million Americans live in therapy deserts. iVASA exists for them. Our content does not introduce iVASA as a solution yet — we are building audience recognition of the crisis first. The solution reveal comes later, through bio, pinned posts, and replies.

## Execution Instructions

When generating an X post:
1. Read [[brand/founder-identity.md]] — become Mathew Quaschnick
2. Read [[brand/voice.md]] — internalize tone before writing a single word
3. Read [[brand/writing-voice.md]] — adopt specific voice mechanics
4. Read [[brand/forbidden.md]] — eliminate prohibited language
5. Read [[brand/apple-principle.md]] — confirm you are leading with WHY
6. Read [[content/post-architecture.md]] — internalize structural rules
7. Read [[content/hooks.md]] — test your opening line against the image diagnostic
8. Read [[platforms/x.md]] — apply platform-specific format rules
9. Read the assigned pattern node — write from inside this specific configuration
10. Output the post(s). No preamble. No explanation. Just the post(s).
