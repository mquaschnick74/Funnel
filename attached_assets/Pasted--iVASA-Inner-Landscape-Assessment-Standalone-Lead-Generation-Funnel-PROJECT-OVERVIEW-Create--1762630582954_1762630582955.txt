# iVASA Inner Landscape Assessment - Standalone Lead Generation Funnel

## PROJECT OVERVIEW

Create a standalone single-page web application for lead generation that runs BEFORE user signup. This is a pre-signup assessment quiz that:
1. Captures leads through an engaging 5-question "Inner Landscape" assessment
2. Generates a personalized therapeutic profile
3. Shows partial results as a "hook" to drive signups
4. Redirects to the main iVASA app at beta.ivasa.ai with profile data

This is NOT the post-signup onboarding - this is a marketing funnel to acquire new users.

---

## TECH STACK

**Simple & Fast:**
- Single HTML file with embedded React (via CDN)
- Vanilla CSS for styling
- No build process (instant loading critical for ads)
- Client-side only (no backend needed initially)
- Responsive mobile-first design

**OR if you prefer a framework:**
- Vite + React + TypeScript
- Tailwind CSS
- Single route application

**Choose whichever you think will be fastest to load and easiest to maintain.**

**Question 1:**
"When you think about your anxiety, it feels like..."

**Choices:**

- A fog bank - can't see clearly, unsure what's ahead

**Question 2:**
"When facing two choices that both feel impossible, you typically..."

**Choices:**
- Stand frozen between them - unable to choose because both feel right and wrong
- Rush down one path, then panic and run back - keep switching, never committing
- Try to build a bridge between them - maybe you can combine them somehow
- Sit down in the middle - you'll wait until you know for sure
- Close your eyes and let your body choose - see what happens
- Split yourself in two to walk both - you'll just do everything

**Question 3:**
"In your daily life, worry or anxiety feels..."

**Choices:**
- Almost constant - like perpetual storms, the weather is always turbulent
- Sudden and intense - calm, then BOOM, overwhelming, then calm again
- A persistent gray cloud - not dramatic, but the sun hasn't broken through in a while
- Completely unpredictable - you never know what's coming, which makes it worse
- Occasional but intense - rare, but you want to understand and prevent it

**Question 4:**
"The idea of truly resting - like actually stopping and being still - feels..."

**Choices:**
- Impossible - you can't rest until everything is handled
- Dangerous - if you stop, something bad will happen
- Guilty - rest feels like failure or selfishness
- Confusing - you don't know how to rest anymore
- Uncomfortable, but doable - you can sit with the discomfort
- Your body won't let you - too much activation to settle

**Question 5:**
"If things could shift - if your relationship with anxiety could change - you'd most want..."

**Choices:**
- Your body to feel calm and settled - the ground beneath you steady
- Emotions to pass through like weather - they come and go, but don't control you
- Anxiety to walk beside you quietly - still there, but not consuming
- To make decisions without panic - paths don't feel like traps
- To simply breathe - fundamental, but everything
- All of these - the whole landscape feels different

## SIGNUP REDIRECT & DATA PASSING

When user clicks "Create Free Account":

**Option A: URL Parameters (Simple):**
https://beta.ivasa.ai/signup?source=assessment&profile=[base64-encoded-json]

Encode their answers and computed profile:
```javascript
const profileData = {
  q1: selectedAnswers[1],
  q2: selectedAnswers[2],
  q3: selectedAnswers[3],
  q4: selectedAnswers[4],
  q5: selectedAnswers[5],
  pattern: computedPattern,
  metaphor: computedMetaphor,
  timestamp: Date.now()
};

const encoded = btoa(JSON.stringify(profileData));
window.location.href = `https://beta.ivasa.ai/signup?source=assessment&profile=${encoded}`;
```

**Option B: LocalStorage (Cleaner URL):**
```javascript
localStorage.setItem('assessment_profile', JSON.stringify(profileData));
window.location.href = 'https://beta.ivasa.ai/signup?source=assessment';
```

Main app reads from localStorage after signup and stores in database.

**Recommendation:** Use Option A (URL params) - more reliable across domains and devices.

## ANALYTICS & TRACKING

**Events to Track:**
1. `page_view` - Landing page loaded
2. `assessment_started` - Clicked "Start Assessment"
3. `question_1_answered` - Completed first question
4. `question_2_answered` - etc.
5. `assessment_completed` - Finished all 5 questions
6. `results_viewed` - Saw their profile
7. `signup_clicked` - Clicked create account button
8. `email_captured` - Provided email as backup

**Implementation:**
Use simple fetch calls to your analytics endpoint:
```javascript
function trackEvent(eventName, properties = {}) {
  fetch('/api/track', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      event: eventName,
      properties: properties,
      timestamp: Date.now(),
      sessionId: getSessionId()
    })
  }).catch(err => console.log('Analytics error:', err));
}
```

Or use Google Analytics, Mixpanel, or PostHog if you prefer.

## DESIGN SPECIFICATIONS

**Color Palette:**
- Background: `#0f0f1a` (dark, almost black)
- Primary accent: `#10b981` (emerald green)
- Text: `#ffffff` (white for headings), `#a0a0b0` (light gray for body)
- Borders: `#10b981` at 30% opacity
- Buttons: Emerald green with glow effect on hover

**Typography:**
- Headings: Inter or System UI, bold, large (32-48px)
- Body: Inter or System UI, normal, readable (16-18px)
- Line height: 1.6 for readability

**Spacing:**
- Generous whitespace
- Mobile-first: 16px padding minimum
- Desktop: 24-32px padding

**Animations:**
- Smooth transitions (300ms ease)
- Subtle fade-ins on question changes
- Hover states on buttons (scale + glow)
- Loading animations feel premium

**Mobile Responsiveness:**
- Single column layout on mobile
- Touch-friendly button sizes (min 44px height)
- Questions stack vertically
- Progress bar always visible at top
